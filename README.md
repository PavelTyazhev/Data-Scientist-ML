# Платформа «Умный реестр подписок» (Smart Registry of Subscriptions)
### Тестовое задание: Разработчик ИИ-агентов

Данный репозиторий содержит полноценное full-stack решение для бизнес-кейса **Умного реестра подписок** с интегрированным **ReAct (Reasoning & Acting) ИИ-агентом**, который умеет отвечать на сложные вопросы пользователя, вызывая инструменты и агрегируя финансовые данные.

---

### 1. Инструкция для проверяющего (Локальный запуск)

#### Требования
* Установленный **Node.js** (версии 18 или новее)
* Установленный менеджер пакетов **npm** (идет вместе с Node.js)

#### Пошаговый запуск
1. **Клонирование репозитория:**
   ```bash
   git clone https://github.com/ВАШ_НИКНЕЙМ/ВАШ_РЕПОЗИТОРИЙ.git
   cd ВАШ_РЕПОЗИТОРИЙ
   ```

2. **Настройка переменных окружения:**
   В корне проекта создайте файл `.env` (вы можете скопировать настройки из `.env.example`):
   ```bash
   cp .env.example .env
   ```
   Откройте созданный файл `.env` и укажите ваш токен авторизации GigaChat (или оставьте значение по умолчанию, так как в коде прописан работающий резервный токен для удобства проверки):
   ```env
   # Токен авторизации Сбербанк GigaChat (OAuth клиентский секрет)
   GIGACHAT_TOKEN="ВАШ_КЛИЕНТСКИЙ_СЕКРЕТ_GIGACHAT"
   APP_URL="http://localhost:3000"
   ```

3. **Установка зависимостей:**
   ```bash
   npm install
   ```

4. **Запуск в режиме разработки:**
   ```bash
   npm run dev
   ```
   После этого откройте в браузере страницу **`http://localhost:3000`**. Вы получите полноценно функционирующее приложение с работающим ReAct-агентом и симуляцией финансовых обязательств.

5. **Сборка и запуск в Production (опционально):**
   ```bash
   npm run build
   npm start
   ```
   Сборка скомпилирует клиентскую часть React и объединит серверный TypeScript-код в производительный файл `dist/server.cjs` с помощью `esbuild`.

#### Запуск с помощью Docker (Альтернативный способ)
Проект полностью контейнеризирован и может быть легко запущен с помощью Docker и Docker Compose.

**Вариант А. Запуск через Docker Compose (рекомендуется):**
1. Убедитесь, что вы настроили файл `.env` (создали его из `.env.example`).
2. Запустите сборку и старт контейнеров:
   ```bash
   docker-compose up --build
   ```
3. Приложение будет доступно по адресу **`http://localhost:3000`**. Папка базы данных подписок (`src/data`) смонтирована как volume, поэтому все ваши изменения сохранятся на хост-машине.

**Вариант Б. Запуск напрямую через Docker CLI:**
1. Соберите Docker-образ:
   ```bash
   docker build -t smart-subscription-app .
   ```
2. Запустите контейнер:
   ```bash
   docker run -d -p 3000:3000 --env-file .env -v $(pwd)/src/data:/app/src/data --name subscription-app smart-subscription-app
   ```

## 2. Выбор LLM и обоснование

Для работы ReAct-агента была выбрана модель **`GigaChat` (от Сбера)**. 

**Почему именно она:**
1. **Финтех-направленность (Sberbank Ecosystem):** Разработка ведется для финтех-задачи (контроль финансовых обязательств). GigaChat изначально обучался с упором на русскоязычную финансовую и банковскую терминологию, отлично оперирует валютами, датами, понятиями списаний и банковских тарифов на локальном рынке.
2. **Высокое качество понимания русскоязычных промптов:** Модель превосходно понимает сложные системные инструкции на русском языке, стабильно удерживая логическую цепочку рассуждений в ReAct-цикле (`thought` -> `action` -> `observation` -> `finalAnswer`).
3. **Нативная поддержка JSON-режима (Structured Outputs):** Мы используем параметр `response_format: { type: 'json_object' }` при вызове GigaChat API. Это гарантирует возврат строго структурированного JSON, что обеспечивает 100% стабильность парсинга мыслей и действий ИИ на каждой итерации и исключает падения из-за ошибок синтаксиса или лишнего текста в ответе.
4. **Безопасность данных (Data Privacy):** Использование отечественной большой языковой модели от Сбера соответствует лучшим финтех-практикам суверенности и безопасности обработки данных клиентов.
5. **Превосходный баланс скорости и логики:** Модель мгновенно переключается между использованием системных инструментов (получение подписок, конвертация валют) и математическими вычислениями, сводя задержку ответа в чате к минимуму.

## 3. Пример полного ReAct-трейса рассуждений

При отправке пользователем запроса:  
**«Сколько я потрачу в ближайшие 30 дней? Покажи итог в рублях.»**

ИИ-агент выполняет следующую цепочку шагов, логируемую в консоли сервера и интерактивно отображаемую в интерфейсе:

### **Шаг 1**
* **Thought**: Пользователь хочет узнать общую сумму расходов в рублях за ближайшие 30 дней. Текущая дата 15 июля 2026 года. Мне нужно сначала получить все финансовые обязательства пользователя с активным статусом. Я вызову инструмент `get_obligations` с фильтром `status: "active"`.
* **Action**: `get_obligations(status: "active", category: null)`
* **Observation**: *(Возвращает массив из 11 активных подписок: Netflix (9.99 USD), Electricity Bill (4500 RUB), Yandex Plus (399 RUB), Adobe (29.99 EUR) и др.)*

### **Шаг 2**
* **Thought**: Теперь мне нужно отфильтровать подписки, у которых дата следующего платежа (`next_payment_date`) находится в диапазоне от 2026-07-15 до 2026-08-14 включительно.
  - Netflix Premium (2026-07-20) - Да, 9.99 USD.
  - Electricity Bill (2026-07-18) - Да, 4500 RUB.
  - Yandex Plus (2026-07-16) - Да, 399 RUB.
  - AWS Cloud (2026-08-01) - Да, 45.50 USD.
  - Gym Membership (2026-07-25) - Да, 3500 RUB.
  - Adobe Creative Cloud (2026-07-22) - Да, 29.99 EUR.
  - ChatGPT Plus (2026-07-29) - Да, 20.00 USD.
  - House Insurance (2026-08-12) - Да, 120.00 EUR.
  - Internet (Rostelecom) (2026-07-19) - Да, 800 RUB.
  - GitHub Copilot (2026-07-21) - Да, 10.00 USD.
  - Spotify Family (2026-08-05) - Да, 14.99 USD.
  Все активные подписки попадают в этот период! Теперь я должен конвертировать суммы в RUB. Начну по очереди. Переведем сначала все долларовые подписки (USD): Netflix (9.99), Spotify (14.99), AWS (45.50), ChatGPT (20.00), GitHub (10.00). Суммарно в USD: 9.99 + 14.99 + 45.50 + 20.00 + 10.00 = 100.48 USD. Вызову `convert_currency` для этой суммы.
* **Action**: `convert_currency(amount: 100.48, from: "USD", to: "RUB")`
* **Observation**: `{"amount": 9093.44, "source": "fallback", "rate": 90.5}`

### **Шаг 3**
* **Thought**: Теперь переведем подписки в евро (EUR): Adobe (29.99 EUR) и House Insurance (120.00 EUR). Суммарно в EUR: 29.99 + 120.00 = 149.99 EUR. Вызову `convert_currency` для этой суммы.
* **Action**: `convert_currency(amount: 149.99, from: "EUR", to: "RUB")`
* **Observation**: `{"amount": 14729.02, "source": "fallback", "rate": 98.2}`

### **Шаг 4**
* **Thought**: Теперь сложим все расходы, которые изначально были в рублях (RUB):
  - Electricity Bill: 4500 RUB
  - Yandex Plus: 399 RUB
  - Gym Membership: 3500 RUB
  - Internet (Rostelecom): 800 RUB
  Итого в RUB изначально: 4500 + 399 + 3500 + 800 = 9199 RUB.
  Теперь сложим все части:
  - Сумма из USD: 9093.44 RUB
  - Сумма из EUR: 14729.02 RUB
  - Сумма из RUB: 9199.00 RUB
  Общий итог: 9093.44 + 14729.02 + 9199 = 33021.46 RUB.
  У меня есть все данные для детального финального ответа.
* **Final Answer**: «В ближайшие 30 дней (с 15 июля по 14 августа 2026 года) вы потратите в общей сложности **33 021,46 рубля**.

  Вот детальная разбивка ваших расходов с учетом конвертации валют:
  1. **Списания в RUB (всего 9 199,00 ₽)**:
     * Electricity Bill — 4 500 ₽ (18 июля)
     * Yandex Plus — 399 ₽ (16 июля)
     * Gym Membership — 3 500 ₽ (25 июля)
     * Internet (Rostelecom) — 800 ₽ (19 июля)
  2. **Списания в USD (всего 100,48 $, переведено как 9 093,44 ₽ по курсу 90.50)**:
     * Netflix Premium — 9,99 $ (20 июля)
     * Spotify Family — 14,99 $ (5 августа)
     * AWS Cloud — 45,50 $ (1 августа)
     * ChatGPT Plus — 20,00 $ (29 июля)
     * GitHub Copilot — 10,00 $ (21 июля)
  3. **Списания в EUR (всего 149,99 €, переведено как 14 729,02 ₽ по курсу 98.20)**:
     * Adobe Creative Cloud — 29,99 € (22 июля)
     * House Insurance — 120,00 € (12 августа)

  Все ваши активные подписки попадают в ближайшие 30 дней. Неактивный тариф "Mobile Plan (Tele2)" в расчете не учитывался.»

## 4. Ограничения агента (Что ИИ-агент не умеет)

Хотя агент является высокоточным логическим ядром, в текущей версии заложены архитектурные границы контроля:
1. **Отсутствие прямого изменения базы данных через чат:** Агент умеет только читать (`get_obligations`) и конвертировать валюты. Он не может изменять, создавать или удалять подписки из базы данных посредством текстовых команд. Это спроектировано для предотвращения несанкционированного изменения финансовых записей (Security-by-Design).
2. **Ограничение на количество итераций (Max 8):** В случае зацикливания или получения некорректных данных от сторонних источников, агент прервет выполнение, сообщив об ошибке, что исключает бесконечные циклы вызовов LLM.
